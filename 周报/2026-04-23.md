### MV Agent 原子化工具与对话架构调研（1148 行，新增）

**为什么做：** MV Agent 从原型推进到可落地架构前，需要明确工具分层、意图识别、画布同步和对话调度方案。

- **三层工具机制**：定义 Tool（单步确定性操作）/ Skill（多步骤工作流）/ MCP（外部服务集成）的分层标准，明确各层定位和边界。
- **8 个原子工具清单**：`analyze_music`、`generate_image`、`generate_text`、`upload_asset`、`evaluate_quality`、`web_search`、`select_model`、`check_timeline`。每个工具定义输入输出 Schema、触发条件、耗时预估和画布事件映射。
- **伪原子 → 真原子改造**：将 `generate_proposals` 黑盒 Skill 拆为 `generate_text × 3 + generate_image × N`，解决用户无法干预中间结果的问题。`generate_image` 从一次 count=N 改为一次 1 张、Agent 并行调度。
- **组合 Skill 定义**：`generate_proposals`、`mv_planner`、`mv_generator`、`video_generation` 作为原子工具的固定编排，不直接实现功能。
- **工具注册与发现机制**：设计 `ToolRegistry` Schema（input/output schema、依赖声明、性能特征、灰度发布标记），支持语义意图匹配和版本管理。
- **意图决策树 + LangGraph StateGraph**：基于现有 chat_agent 意图系统，扩展 12 种 MV 创作意图路由，实现 detect_intent → route_by_intent → 选择工具链的条件路由架构，支持并行工具调用（Send API 扇出）。
- **画布同步协议**：定义 12 种 CanvasEvent 类型（music_analysis_ready / asset_created / video_clip_ready 等）和 4 种反向 CanvasActionEvent，实现对话面板与无限画布的双向联动。
- **流程内引导 + 自由跳入兼容**：双状态模型（pipeline_stage 仅作进度提示，不作为准入条件），工具前置条件表（零前置 / 可自动补齐 / 必须确认三类），支持"生成一张角色图"随时调用和"开始生成视频"自动补齐整条链路。
- **回退三级策略**：重试同模型 → 换模型 → 降级输出，确保任何失败都不让用户看到空白。
- **参考现有 Skill 架构**：提取 mv-planner / mv-generator 的 Lazy Loading References、Route-and-Dispatch、Fallback by Design、Output Discipline 等可复用模式。

### MV Agent 全貌文档（653 行，新增）

**为什么做：** 调研报告定义工具架构后，需要一份"一张表看懂所有东西"的全景参考，涵盖工具、交互、架构和 PRD 约束。

- **全景分类表**：8 个原子工具 + 4 个 Skill 组合 + 4 个 MCP Server + 若干 Agent 内部能力 + 6 种对话回复格式 + 13 种画布事件 + 6 类用户意图 + 7 个流程阶段 + 3 级回退策略。
- **原子工具细化（8 个）**：在调研报告基础上新增 `generate_video`（异步 60-120s）、`get_asset`（读存储 <1s）、`push_to_canvas`（前端同步 <1s）。每个工具定义输入输出、耗时、画布事件映射和外部依赖。
- **Agent 内部能力澄清**：明确意图识别、文字生成、方案构思、模型选择、时间轴校验、前置检查、回退决策、对话回复生成等**不需要注册为工具**——Agent 本身就是 LLM，这些是其天然能力。
- **伪原子纠正**：`upload_asset` 归为前端操作（用户上传，非 Agent 调用）；`generate_text`、`select_model`、`check_timeline` 归为 Agent 内部能力（不涉及外部调用）。
- **4 个组合 Skill 编排**：`generate_proposals`（Agent 构思 + generate_image×N）、`mv_planner`（纯 Agent 构思，无需工具）、`mv_generator`（纯 Agent 构思）、`video_generation`（mv_generator + generate_video×N）。
- **6 种对话回复格式**：TextReply、AssetCard（缩略图+操作按钮）、ProposalCard（摘要+选择按钮）、OptionCard（问题+可点击选项）、ProgressIndicator（N/M 进度）、LoadingState。
- **13 种画布事件**：10 种正向同步（music_analysis_ready → canvas_focus）+ 5 种反向同步（element_selected → preview_requested），定义事件类型、触发工具、目标区域和画布响应。
- **6 类用户意图**：自由操作（零前置）、流程推进（下一步）、链式执行（多步配合）、反馈修改（不满意）、状态查询（问进度）、闲聊。
- **7 个流程阶段**：idle → music_analysis → proposals → assets → storyboard → video_generation → review，仅作进度标记，不作为准入条件。
- **4 个完整交互流程示例**：线性全流程、自由跳入、问题选项完整流程、画布选中元素→Agent 修改。
- **Agent 核心循环**：意图识别 → 前置检查 → 执行（内部推理+外部工具混合）→ 推送画布事件 → 更新 State → 生成对话回复。
- **PRD 产品约束**：对话面板 360px 固定宽度、OKLCH 琥珀橙强调色、动画规范（150ms~1.2s）、[@xxx] 双向引用标签、5 种人格系统（专业导演/创意伙伴/温柔助手/毒舌审美/艺术家）。
- **架构总览图**：前端层（360px 对话面板 + 无限画布）→ Agent 编排层（内部能力 + 原子工具 + Skill 组合 + MCP）→ 工具服务层。
