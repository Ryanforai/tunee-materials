# Cover 功能上线讨论

**日期：** 2026-04-24

**状态：** 讨论中

---

## 1. 功能概述

用户在同一次聊天消息中**同时上传音乐文件 + 自然语言描述风格**（如发一个音频文件并附言"来个爵士版的"），系统识别风格后生成 Cover 版本：
- **AI 翻唱**：保留原歌词，按指定风格翻唱
- **AI 二创**：基于原曲结构生成全新歌词 + 按指定风格生成新音乐

---

## 2. 核心流程

```
用户发送：音乐文件 + 自然语言风格描述
  │
  ├── 风格意图识别 → 提取风格关键词，写入 music prompt
  │
  ├── 识别原曲信息
  │   ├── 段落识别模型 → verse/chorus/bridge/intro/outro 等
  │   ├── ASR 识别歌词
  │   └── 联网搜索原曲歌词校对
  │
  ├── 有用户上传歌词？
  │   ├── 是 → 使用用户提供的歌词（跳过歌词生成）
  │   └── 否 → 走歌词生成流程：
  │       ├── 整理校对后的歌词，计算每行字数
  │       ├── 按段落结构 + 字数约束生成新歌词
  │       └── 输出与原曲结构匹配的新歌词
  │
  └── 生成音乐
      ├── 歌词（用户提供的 / 新生成的）
      ├── music prompt（含识别到的风格描述）
      └── 自研模型生成音乐
      └── 输出最终 Cover 版本（只输出一种）
```

---

## 3. 歌词生成规则（当用户未提供歌词时）

- **段落类型覆盖：** Verse / Chorus / Bridge / Intro / Outro 等全部类型
- **字数约束：** 每行生成歌词字数 ≤ 原曲对应行字数
- **结构约束：** 新歌词段落编排与原曲一致

---

## 4. 异常处理设计

### 4.1 ASR 识别失败

| 场景 | 处理方式 |
|------|----------|
| ASR 未识别到歌词（纯音乐/乐器曲） | 联网查原曲；若也搜不到，直接走风格翻唱（不生成歌词，纯音乐改编） |
| ASR 识别歌词但不准确 | 联网校对补全，以联网结果为准；联网也搜不到，以 ASR 结果为准并标注「歌词可能不准确」提示 |
| ASR + 联网都失败，但段落识别成功 | 基于段落结构生成全新歌词（不依赖原歌词） |
| ASR + 联网 + 段落识别都失败 | 返回友好提示：「暂时无法处理这段音频，请尝试上传其他音乐」 |

### 4.2 歌词生成失败

| 场景 | 处理方式 |
|------|----------|
| 字数约束无法满足（某行字数过少，如 ≤ 2 字） | 该段落改为哼唱/器乐段落，不强制生成歌词 |
| 生成模型超时/报错 | 重试一次；仍失败则降级为纯音乐翻唱（不生成新歌词） |

### 4.3 音乐生成失败

| 场景 | 处理方式 |
|------|----------|
| 自研模型生成失败/超时 | 重试一次；仍失败返回提示：「生成遇到一些问题，请稍后重试」 |
| 生成音乐时长严重偏差 | 截断或填充处理，保证输出合理时长 |

### 4.4 其他异常

| 场景 | 处理方式 |
|------|----------|
| 上传文件格式不支持 | 上传时即时校验，返回格式提示 |
| 上传音频过长导致内存/超时 | 不设硬限制，但后台监控；极端超长音频（如 > 30 分钟）提示「目前暂不支持超长音频」 |
| 联网搜索服务不可用 | 降级为仅使用 ASR + 段落识别结果，功能不中断 |
| 风格未指定/无法识别 | 尝试兜底识别；仍无法识别则提示用户补充或默认使用原曲风格 |

## 5. 技术方案

### 5.1 前端
- 聊天内嵌式：用户在聊天中发送音乐文件，自动触发 Cover 流程
- 生成中：显示进度状态（预计 2~5 分钟）
- 生成完成：展示 Cover 结果（音频播放 + 歌词展示）

### 5.2 后端 / 服务
- 段落识别模型
- ASR 服务
- 联网查原曲歌词（外部搜索服务）
- 歌词生成模型
- 音乐生成模型（自研）
- 自研模型生成音乐

### 5.3 依赖项
- ASR 服务可用性
- 联网搜索歌词服务
- 自研音乐模型性能
- 歌词生成模型性能

## 8. 埋点 & 指标

- **入口指标：** Cover 功能使用率（聊天中上传音乐触发 Cover 的占比）
- **漏斗指标：** 上传成功率 → 识别成功率 → 歌词生成成功率 → 音乐生成成功率 → 最终完成率
- **质量指标：** 歌词准确率（ASR+联网校对后）、生成音乐完成率
- **异常指标：** 各环节失败率、重试率、超时率
- **用户行为指标：** Cover 播放完成率、重复使用率
