# MV Agent 产品决策

> 服务于音乐创作者 / AI音乐人 / MV兴趣小白 —— 将音乐视觉化

---

## 产品定义

- **交互范式**：左对话 + 右画布。对话承接意图，画布承接结果
- **对标**：Lovable / Lovart，专注 MV 细分领域
- **差异化**：音乐深度分析 → 方案渐进确认 → 画布统一呈现（非 tab 切换）

### 8 个核心 Agent 职责

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **对话 Agent** | 意图识别、对话状态机、引导追问 | 用户消息 | 回复 + 操作指令 |
| **音乐分析 Agent** | 音频分析、歌词解析、情绪提取 | 音频文件 | 分析报告 |
| **创意策划 Agent** | 基于分析结果生成 MV 方案 | 分析报告 + 用户偏好 | 2-3 个方案 |
| **资产管理 Agent** | 引导创建/上传/编辑资产 | 方案描述 / 用户上传 | 资产列表 |
| **分镜规划 Agent** | 将方案转化为分镜脚本 | 选定方案 + 资产 | 分镜 JSON |
| **视频生成 Agent** | 调用视频模型生成分段视频 | 分镜 + 资产 | 视频片段 |
| **质量评估 Agent** | 自动审核 + 用户反馈收集 | 生成结果 | 评分 + 改进建议 |
| **状态管理 Agent** | 维护创作流程状态、依赖图谱 | 所有模块 | 状态快照 + 版本历史 |

## 核心决策

### 1. 对话模式 —— 纯意图识别，不暴露给用户

Agent 根据输入自动判断意图（创作/闲聊/灵感/搜索），用户无感知。不设手动切换，降低认知负担。

### 2. 画布 —— 统一画布，全部内容纵向排列

不用 tab 切换。音乐分析、方案、资产、分镜、预览全部在同一画布上从上到下排列，用户滚动浏览和编辑。减少信息碎片化，让用户对创作全貌有直观感知。

### 3. 模型/参数 —— ⚙️ 按钮进入，默认 Auto

对话框底部有模型标签 + 设置按钮。全部默认 Auto，高级用户可手动指定图片/视频/对话模型。

### 4. 画布 ↔ 对话双向引用

画布内容可引用回对话框实现精准编辑：
- 点击方案/分镜卡片 → 插入 `[@对象]` 引用标签
- 框选画布区域 → 截图附到对话框
- 时间轴锚点 → 自动生成时间戳引用

### 5. 多模态上传 —— 📎 按钮

支持图片/视频/音频/文本上传。Agent 自动判断用途（参考级：提取风格 / 素材级：直接使用），不确定时询问用户。

### 6. 记忆系统 —— 四级

| 层级 | 范围 | 内容 |
|------|------|------|
| 用户偏好 | 跨项目永久 | 风格偏好、常用素材、人格设置 |
| 项目记忆 | 同项目内 | 创意决策、修改历史、原因 |
| 会话记忆 | 当前会话 | 对话历史、画布状态 |
| 工作记忆 | 当前任务 | 编辑焦点 |

### 7. 人格系统 —— 5种预设 + 自定义

| 人格 | 创作倾向 | 适合 |
|------|----------|------|
| 专业导演 | 经典电影语言 | 有经验用户 |
| 创意伙伴 | 实验性方案 | 小白用户 |
| 温柔助手 | 保守稳妥 | 需要引导的用户 |
| 毒舌审美 | 高品质高难度 | 追求品质的用户 |
| 艺术家 | 诗意表达 | 追求氛围感的用户 |

人格不只是说话方式，创作建议风格也会随之不同。

## 用户流程

```
上传音乐 → 音乐分析（情绪曲线+关键词）
         → 出方案（2-3个方向，画布展示）
         → 选方案 → 引导资产创作
         → 生成分镜 → 生成视频
         → 随时预览 → 导出成片
```

每个阶段都可以在画布上直接编辑或通过对话修改，支持随时回溯上游并重新生成下游。

---

## MVP 优先级

| 优先级 | 内容 | 原因 |
|--------|------|------|
| P0 | 音乐分析 + 方案生成 | 核心价值点 |
| P0 | 对话系统（意图识别） | 交互载体 |
| P0 | 统一画布 | 视觉呈现 |
| P1 | 资产管理 + 分镜规划 | 从方案到执行 |
| P2 | 视频生成 | 成本高，验证后接 |
