# MV Agent 产品决策

> 服务于音乐创作者 / AI音乐人 / MV兴趣小白 —— 将音乐视觉化

---

## 产品定义

- **交互范式**：左对话 + 右画布。对话承接意图，画布承接结果
- **对标**：Lovable / Lovart，专注 MV 细分领域
- **差异化**：音乐深度分析 → 方案渐进确认 → 画布统一呈现（非 tab 切换）

### 1. 架构模式 —— 主 Agent + 工具/Skill，非多 Agent 编排

原设计 8 个 Agent 改为 **1 个主 Agent + 工具集**：

| 原 Agent | 新形态 | 说明 |
|----------|--------|------|
| 对话 Agent | **主 Agent 本体** | 意图识别 + 状态机 + 工具调度 |
| 音乐分析 Agent | Tool: `analyze_music` | 音频分析 → 情绪曲线/关键词/BPM |
| 创意策划 Agent | Tool: `generate_proposals` | 基于分析结果生成 2-3 个 MV 方案 |
| 资产管理 Agent | Tool: `manage_assets` | 创建/上传/编辑视觉资产 |
| 分镜规划 Agent | **Skill: `mv-planner`** | 复用现有 视频skill/ 中的 mv-planner |
| 视频生成 Agent | **Skill: `mv-generator`** | 复用现有 视频skill/ 中的 mv-generator |
| 质量评估 Agent | Tool: `evaluate_quality` | 自动审核 + 改进建议 |
| 状态管理 Agent | **内置主 Agent** | 依赖图谱、状态快照 |

**现有 Skill 基础设施（已验证）**：
- `视频skill/` 目录已包含成熟的 Skill 架构（mv-planner + mv-generator 双节点版）
- Skill 自含路由引擎：frontmatter 触发、懒加载引用、内部分发逻辑
- 9 维度提示词系统（物理维度、镜头语言等）已沉淀为可复用模块
- `creative_analysis/` 提供剧本大纲节点 Prompt，可作 Skill 内部调用

**三层工具机制**：

| 层级 | 用途 | 示例 |
|------|------|------|
| **Tool（LangGraph Node）** | 快速、确定性的单步操作 | 音频分析、质量评估、文件操作 |
| **Skill（多步骤流程）** | 复杂、多步骤的工作流 | mv-planner（分镜规划）、mv-generator（视频生成） |
| **MCP Server** | 外部服务集成 | 模型 API 路由、存储、第三方服务 |

**主 Agent 架构（LangGraph StateGraph）**：

```
User Input → 意图识别 → 选择工具/Skill → 执行 → 更新状态 → 渲染右侧面板
                                       ↓
                                 自动/并行调用多个工具
                                       ↓
                                 汇总结果 → 回复用户
```

- **状态驱动**：用 LangGraph StateGraph 维护创作流程状态（分析 → 方案 → 资产 → 分镜 → 视频）
- **条件路由**：根据用户意图和当前状态自动路由到对应工具
- **支持回退**：用户可随时回溯上游阶段并重新生成下游

### 2. 对话模式 —— 纯意图识别，不暴露给用户

Agent 根据输入自动判断意图（创作/闲聊/灵感/搜索），用户无感知。不设手动切换，降低认知负担。

### 4. 模型/参数 —— ⚙️ 按钮进入，默认 Auto

对话框底部有模型标签 + 设置按钮。全部默认 Auto，高级用户可手动指定图片/视频/对话模型。

### 5. 画布 ↔ 对话双向引用

面板内容可引用回对话框实现精准编辑：
- 点击方案/分镜卡片 → 插入 `[@对象]` 引用标签
- 框选面板区域 → 截图附到对话框
- 时间轴锚点 → 自动生成时间戳引用

### 6. 多模态上传 —— 📎 按钮

支持图片/视频/音频/文本上传。Agent 自动判断用途（参考级：提取风格 / 素材级：直接使用），不确定时询问用户。

### 7. 记忆系统 —— 四级

| 层级 | 范围 | 内容 |
|------|------|------|
| 用户偏好 | 跨项目永久 | 风格偏好、常用素材、人格设置 |
| 项目记忆 | 同项目内 | 创意决策、修改历史、原因 |
| 会话记忆 | 当前会话 | 对话历史、面板状态 |
| 工作记忆 | 当前任务 | 编辑焦点 |

### 8. 人格系统 —— 5种预设 + 自定义

| 人格 | 创作倾向 | 适合 |
|------|----------|------|
| 专业导演 | 经典电影语言 | 有经验用户 |
| 创意伙伴 | 实验性方案 | 小白用户 |
| 温柔助手 | 保守稳妥 | 需要引导的用户 |
| 毒舌审美 | 高品质高难度 | 追求品质的用户 |
| 艺术家 | 诗意表达 | 追求氛围感的用户 |

人格不只是说话方式，创作建议风格也会随之不同。

### 9. Web 工具集成 —— 自动搜索/抓取参考，补充非补充核心生成

| 工具 | 触发场景 | 输出 |
|------|----------|------|
| **WebSearch** | 用户搜索参考/灵感不足时自动触发 | MV 参考列表 + 缩略图 + 链接 |
| **WebFetch** | 用户粘贴 URL 后自动抓取内容 | 页面内容摘要 + 视觉元素提取 |

**应用场景**：
- 搜索类似风格 MV 参考（"搜一下王家卫风格的 MV"）
- 抓取 Pinterest/Behance 情绪板作为风格参考
- 自动从 YouTube/Vimeo 链接提取视频关键帧
- 搜索结果以卡片形式展示在右侧面板

**定位**：Web 工具仅补充对话和创意策划能力，**核心视频生成 Skill（mv-planner/mv-generator）完全基于本地引用运行**，不依赖外部网络。LangChain 遵循标准 Tool 注册模式，在 ReAct 循环中调用。

## MVP 优先级
