# MV Workspace 编辑器需求文档（精简版）

> 版本：v1.0 | 日期：2026-04-24 | 状态：待评审 | 修订：PRD 审计后 v1.1
> 原型参考：`mv-editor-simple.html`

---

## 1. 背景与目标

### 1.1 背景

当前 MV 生成流程中，用户在获得 AI 生成的分镜脚本后，缺少一个**可视化的剪辑工作台**来完成素材浏览、片段编排、节奏对齐和最终导出。现有工具（剪映、Premiere）以视觉为核心、操作复杂，不适合音乐创作者快速完成 MV 剪辑。

### 1.2 目标

打造一个**音乐驱动**的 MV 编辑工作台，让用户在一个沉浸式环境中完成「分镜 → 素材编排 → 播放预览 → 导出」的完整链路。

### 1.3 核心差异化定位

| 传统编辑器（剪映/PR） | MV Workspace |
|---|---|
| 视觉驱动，帧率为第一时间基准 | **音乐驱动，BPM 为第一时间基准** |
| 多轨道复杂操作 | 单视频轨 + 单音频轨 + 单字幕轨，降低门槛 |
| 冷色调专业界面 | **浅色主题**，乐谱纸式的工作站氛围 |

### 1.4 目标用户

- 音乐创作者 / AI 音乐人
- 想快速出 MV 的非专业剪辑用户
- 需要节奏对齐的音乐视频制作场景

### 1.5 成功指标

**北极星指标：**
- MV 完成率（至少拖入一个素材到时间轴 + 点击最终导出）

**辅助指标：**
- 平均编辑时长（从进入编辑器到完成导出）
- 素材拖入时间轴次数（人均）
- 撤销/重做使用率（反映编辑深度）

**基线：** 待数据团队在上线后首周采集

**目标：** 上线 4 周内完成率 ≥ 40%，平均编辑时长 ≤ 15 分钟

## 3. 入口与项目上下文

### 3.1 入口路径

- **来源页面：** Tunee MV 平台项目列表页
- **操作：** 点击已有项目的 "Edit" 按钮 → 跳转至 `mv-editor/{project_id}`
- **新建入口：** 项目列表页点击 "New MV" → 触发 AI 分镜脚本生成流程 → 生成完成后自动跳转至编辑器

### 3.2 前置条件

- **登录校验：** 用户必须已登录，未登录则重定向至登录页，登录后自动回跳
- **项目权限：** 仅项目创建者或被授权成员可访问

### 3.3 项目绑定

- **数据隔离：** 所有数据按 `project_id` 隔离，不同项目互不可见
- **单项目模式：** 编辑器同时只处理一个项目，不跨项目操作
- **多项目管理：** 一个用户可拥有多个项目，通过项目列表页切换

### 3.4 首次创建流程

1. 用户在项目列表页点击 "New MV"
2. 填写 MV 基本信息（歌曲、风格、参考等）
3. 后端 AI 生成分镜脚本
4. 生成完成后自动跳转至 `mv-editor/{project_id}`，加载分镜数据

## 4. 端到端 Happy Path

以下是用户从进入页面到完成编辑的完整流程：

### 4.1 进入页面与数据加载

1. 用户从项目列表页点击 "Edit" 进入编辑器
2. 页面初始化：
   - 读取 URL 中的 `project_id`
   - 优先检查 `localStorage` 中是否有 `mv_editor_{project_id}` 的自动保存数据
   - 如有自动保存数据，加载并恢复状态
   - 否则从后端拉取项目分镜数据和素材数据
3. 渲染三栏布局 + TopBar（AI 面板 + 素材面板 + 视频预览 + 时间轴）

### 4.2 浏览素材与拖入时间轴

4. 用户在 Media Panel 中切换分组（All / A / B / C）和类型（Video / Image / Audio）
5. 浏览素材网格，查看缩略图、时长、已使用标识
6. 用户拖拽素材卡片到 Timeline 视频轨
7. 素材自动计算插入位置，渲染为新的片段（Clip）
8. 每次操作触发 `pushHistory` 并自动保存

### 4.3 播放与预览

9. 用户点击播放按钮，时间轴播放头开始移动
10. Video Preview 根据当前播放头所在片段，按三级降级策略展示对应内容
11. 播放头到达片段边界时，预览内容自动切换
12. 用户可通过进度条跳转、倍速控制调节播放

### 4.4 编辑片段

13. 用户在时间轴上拖拽片段调整顺序 → 片段紧凑排列 → 自动保存
14. 用户拖拽片段缩放手柄调整时长 → 实时更新 → 自动保存
15. 用户选中片段后点击 Delete 或按快捷键删除 → 自动保存
16. 误操作可通过 Undo（Cmd/Ctrl+Z）恢复

### 4.5 完成与导出

17. 编辑完成后，用户点击 TopBar 右侧 "Export" 按钮
18. v1 版本弹出 Toast "导出功能开发中..."，后续版本接入实际导出

## 5. 自动保存机制

### 5.1 触发条件

- **每次 `pushHistory` 操作时触发自动保存**
- 具体触发场景：删除片段、分割片段、拖拽片段、导入素材、排序紧凑排列

### 5.2 存储策略

| 版本 | 存储方式 | Key 格式 |
|---|---|---|
| v1 | `localStorage` | `mv_editor_{project_id}` |
| v2（规划） | 服务端同步 + localStorage 双写 | 服务端 API + 本地缓存 |

**v1 存储内容：**
- `CLIPS` 数据（深拷贝）
- `S` 全局状态（`playing`, `t`, `dur`, `zoom`, `sel`, `panel`, `type`, `volume`, `speed`, `subtitleEnabled`）
- 时间戳（用于冲突检测）

### 5.3 防抖节流

- **防抖间隔：** 2 秒（连续操作时，最后一次操作完成后 2 秒才写入）
- 目的：避免高频操作（如拖拽缩放）导致频繁写入

### 5.4 数据恢复

- 页面加载时，优先检查 `localStorage` 中是否有 `mv_editor_{project_id}` 数据
- 如存在且时间戳合理，恢复本地状态（跳过后端拉取）
- 如不存在或数据损坏，从后端拉取最新数据

### 5.5 失败处理

- **写入失败：** TopBar 副标题右侧显示橙色圆点 + "Save failed" 悬浮提示
- **恢复策略：** 用户可点击重试按钮，重新触发保存
- **数据不丢失：** localStorage 写入失败不影响内存中的编辑状态
- **QuotaExceededError 处理**：localStorage 写入失败时捕获错误类型，如为 `QuotaExceededError`，Toast 提示"存储空间不足，自动保存已暂停"，同时保留内存中的编辑状态不丢失。不阻塞用户继续编辑
- **写入失败不阻塞**：所有 `pushHistory` 操作先更新内存状态，再异步触发 localStorage 写入，写入失败不影响当前编辑

### 5.6 未来规划（v2+）

- **服务端同步：** 服务端 API 替代纯 localStorage 存储
- **冲突解决：** 多端编辑时的冲突处理策略
- **多端同步：** 同一项目在不同设备上的编辑状态同步

## 6. 版本规划

### 6.1 版本与文件

| 版本 | 文件 | 说明 |
|---|---|---|
| v1（当前） | `mv-editor-simple.html` | 核心功能完整，面向音乐创作者快速出片 |
| v2+（规划） | 后续迭代 | 多轨道、BPM 网格、AI 生成字幕、实际导出等高级功能 |

### 6.2 版本演进路线

v1 聚焦核心的「素材管理 → 时间轴编排 → 播放预览 → 导出占位」闭环。后续版本在此基础上新增高级功能：

- **v2+：** 多视频/音频轨道、BPM 自动检测与节拍网格、实际 AI 后端接入、实际导出、实际音量/倍速对音频流的影响
- **v2+：** 歌词轨道 + AI 生成字幕（v1 的字幕轨仅展示后端预置文本，不支持自动生成）

## 7. 空状态处理

### 7.1 素材面板空状态

- **场景：** 当前分组/类型下无素材
- **展示：** "No materials yet" 占位文字 + "Import" 按钮
- **交互：** 点击 Import 触发系统文件选择器

### 7.2 时间轴空状态

- **场景：** 视频轨、音频轨、字幕轨均为空
- **展示：** 提示文字仅覆盖视频轨和音频轨区域（字幕轨单独展示为空但不暗示可拖放）。视频轨区域："Drag materials here to start editing"（居中，半透明，0.875rem）；字幕轨区域：显示 "字幕由分镜脚本自动生成"（居中，半透明，0.6875rem）
- **视觉：** 视频轨和音频轨区域显示虚线边框，字幕轨区域显示无框灰色占位文字

### 7.3 预览区空状态

- 已由三级降级策略 Level 3 覆盖：显示音乐结构标签 + 场景描述 + "Video generating" 提示

## 8. 功能详述

### 8.1 TopBar

**布局（从左到右）：**

1. **品牌区**（300px）
   - Logo：26px 圆形，深色背景，白色 "T"
   - 标题："MV Workspace"（0.8125rem, 600）
   - 副标题："Video Editor - Auto-saving"（0.6875rem, 浅色）

2. **浏览器标签**
   - 圆角顶部（7px 7px 0 0），与主背景同色
   - 底部 `margin-bottom: -1px` 重叠底边框
   - 内容：播放图标 + "Video Editor" 文字

3. **右侧操作区**
   - 导出按钮：30px 高，1px 边框，带下载图标
   - 用户头像：30px 橙色圆形，首字母

**交互：**
- 导出按钮：点击弹出 Toast "导出功能开发中..."（v1 为占位，后续接入实际导出）

### 8.2 Media Panel（30%）

#### 8.2.1 分组标签（Primary Tabs）

- All / A / B / C 四组
- 激活态：底部 2px 橙色下划线
- A/B/C 前缀带彩色圆点（A=深色，B=中灰，C=橙色）

#### 8.2.2 类型标签（Type Tabs）

- Video / Image / Audio
- 三等分，大写，0.6875rem，字间距 0.04em
- 激活态：凸起背景 + 1px 边框

#### 8.2.3 素材网格

- 2 列等宽，8px 间距
- 缩略图：16:9 宽高比，OKLCH 暖灰渐变
- 时长徽章（右下角，深色背景白字）
- "已使用"标识（右上角，6px 橙色圆点 + 白环）
- Hover 效果：`translateY(-1px)` + 中等阴影
- **素材覆盖保证**：素材面板中至少包含当前画布时间轴上所有已使用素材。每个在 CLIPS 中出现的素材名称，必须在 MEDIA 的对应分组和类型中存在。
- **素材来源**：初始化时，后端将分镜脚本对应的素材数据写入 MEDIA。用户导入的新素材追加到对应分组。

#### 8.2.4 导入卡片

- 虚线边框 + 上传图标 + "Import Material"
- 点击触发系统文件选择器
- 自动检测文件类型（video/image/audio）并追加到对应分类
- **v1 处理方式**：本地 `FileReader` 读取，不上传服务端。文件以 `Object URL` 形式暂存于内存，刷新页面后丢失（与自动保存策略一致）
- **大小限制**：单文件 ≤ 200MB，超出则 Toast 提示"文件大小超过 200MB 限制"
- **格式支持**：视频（mp4, webm）、图片（png, jpg, webp）、音频（mp3, wav, m4a）。不支持的文件类型：Toast 提示"不支持的文件格式"
- **加载态**：文件选择后显示导入中进度条（0→100%），完成后出现在对应分类网格中
- **失败处理**：读取失败（如文件损坏/被占用）→ Toast "文件读取失败"，不追加到分类

#### 8.2.5 交互

- 切换分组 → 更新 `S.panel` → 重新渲染网格
- 切换类型 → 更新 `S.type` → 重新渲染网格
- 拖拽素材 → HTML5 Drag & Drop，序列化 `{name, dur}` 到 dataTransfer
- **拖放轨道限制**：
  - 视频轨（tr-v1）：接受所有类型的素材拖入
  - 音频轨（tr-au）：仅当轨道为空时接受音频类型素材拖入
  - 字幕轨（tr-sub）：不接受素材拖入（字幕由后端分镜脚本生成）

### 8.3 Video Preview（70%）

#### 8.3.1 预览舞台

- 居中，最大宽度 700px，1px 边框，9px 圆角
- 视频窗口：深色背景，16:9 宽高比，阴影

#### 8.3.2 当前片段标签

- 左上角浮层，0.625rem，半透明背景

#### 8.3.3 悬停播放浮层

- 全屏绝对覆盖，hover 时 opacity 0 → 1
- 居中 54px 磨砂玻璃圆形 + 播放图标

#### 8.3.4 三级降级策略

视频预览根据当前播放头所在片段的选项对象（opts）内容，按以下优先级展示：

| 级别 | 条件 | 展示 | 徽章 |
|---|---|---|---|
| Level 1 | `opts.videoReady === true` | 专属渐变背景（按片段名称匹配不同色调）+ 影片图标 | "视频" |
| Level 2 | `opts.videoReady === false` 但 `opts.imageSeed` 存在 | 基于 seed 生成的渐变模拟分镜图（亮度 0.30+）+ 图片图标 | "分镜图" |
| Level 3 | `videoReady` 和 `imageSeed` 都不存在 | 深色背景 + 音乐结构标签（INTRO/VERSE 等）+ 场景描述 + "视频生成中，以下为分镜描述"提示 | "描述" |

视频轨片段支持三种内容类型：视频、图片、文本。预览区根据片段的 `videoReady` / `imageSeed` / `desc` 字段自动选择降级级别。

无片段时的空状态：显示 "拖拽素材或点击播放" 占位文字 + 影片图标。

#### 8.3.5 播放控制

- **进度条：** 4px 高度（hover 5px），橙色填充，12px 可拖动圆点
- **时间码：** `mm:ss / mm:ss` 格式
- **播放按钮：** 38px 圆形，hover 时 `scale(1.06)`，播放/暂停图标切换
- **音量控制：** Popover 浮层 + 滑块，初始 50%，实时显示百分比
- **倍速控制：** Popover 浮层 + 单选列表（0.8x, 1x, 1.2x, 2x）
- **全屏：** 浏览器 Fullscreen API

### 8.4 Timeline Toolbar（40px）

**左侧工具：**

| 按钮 | 图标 | 功能 |
|---|---|---|
| Delete | X 图标 + 文字 | 删除选中片段 |
| Split | 分割线图标 + 文字 | 在播放头位置分割视频轨片段 |
| 分隔线 | 1px 竖线 | 视觉分隔 |
| Undo | 撤销图标 | 撤销上一步操作（无历史时 disabled） |
| Redo | 重做图标 | 重做上一步操作（无历史时 disabled） |

**快捷键：**
- Delete / Backspace：删除选中片段
- S：在播放头位置分割视频轨片段 + 联动分割字幕轨片段
- Cmd/Ctrl + Z：撤销
- Cmd/Ctrl + Shift + Z：重做
- Space：播放/暂停

**快捷键作用范围：**
- 上述快捷键仅在**非输入框聚焦**时生效。当焦点在任何 textarea/input 内时，这些快捷键走浏览器默认行为
- Space 键仅在时间轴区域聚焦时阻止默认页面滚动

### 8.5 Timeline 时间轴

#### 8.5.1 结构

- 横向滚动容器
- 左侧轨头列：84px 宽，sticky left，z-index 15
- 时间轴内容区：最小宽度 = 时长 × 缩放倍率

#### 8.5.2 轨头（Track Header）

- 48px 高，底部边框
- 左侧 3px 颜色条（绝对定位）：
  - 视频轨：深灰色
  - 音频轨：浅灰色
  - 字幕轨：橙色（`oklch(0.580 0.180 48)`），与强调色一致
- 轨名：居中，0.6875rem，600

#### 8.5.3 标尺（Ruler）

- 28px 高，背景色，底部边框
- 每 15s：主刻度（12px，实线）+ 时间标签
- 每 5s：次刻度（8px，浅色）
- 点击可跳转到对应时间

#### 8.5.4 播放头（Playhead）

- 2px 宽，橙色
- 顶部三角形帽：clip-path polygon，12px 宽，8px 高
- z-index 20，pointer-events none
- 播放时实时更新位置

#### 8.5.5 轨道（Tracks）

精简版包含三条轨道：

| 轨道 | 标识 | 背景 | 内容 |
|---|---|---|---|
| 视频轨 | tr-v1 | 近白色 | 视频片段 + 装饰帧线 |
| 音频轨 | tr-au | 浅灰色 | 音频片段 + Canvas 绘制的波形（确定性伪随机） |
| 字幕轨 | tr-sub | 橙色半透明（0.5 opacity） | 字幕片段 + 虚线边框样式 |

#### 8.5.6 片段（Clips）

**通用样式：**
- 绝对定位，上下 5px 边距，5px 圆角
- 抓取光标，禁止选中文本
- Hover：小阴影
- 选中态（`.sel`）：2px 橙色框阴影 + 中等阴影，z-index 5

**缩放手柄：**
- 两侧各 7px 宽，col-resize 光标
- 伪元素：2px × 14px 竖条，opacity 0.30，hover 0.65
- 左侧拖拽：同时改变 left + width
- 右侧拖拽：仅改变 width
- 最小宽度：20px

#### 8.5.7 片段交互

| 操作 | 触发 | 行为 |
|---|---|---|
| 选中 | 点击片段 | 清除所有 `.sel`，添加当前，更新预览 |
| 拖拽移动 | 中间区域 mousedown + 拖动 | 实时跟随鼠标更新 left 位置。mouseup 后：吸附到 0.5s → 从原位置移除 → 插入新位置 → 排序 → 紧凑排列（无间隙）→ pushHistory → 重渲染。紧凑排列后片段顺序由新位置决定，实现片段间交换效果。紧凑排列仅对视频轨（tr-v1）执行，音频轨（tr-au）保持用户手动放置的位置不变。拖拽过程中不做紧凑排列（实时预览片段位置），仅在 drag-end（mouseup）后触发 |
| 拖拽缩放 | 手柄 mousedown + 拖动 | 左/右侧分别处理，最小 20px |
| 删除 | 点击 Delete / 按键 | 从 CLIPS 中 splice，DOM 移除，清除选中 |
| 分割 | 点击 Split / S 键 | 查找跨越播放头的视频轨片段，一分为二；同时查找跨越播放头的字幕轨片段并一分为二，保持字幕对齐 |
| 拖入上轨 | 从素材网格拖放到轨道任意位置 | 根据 drop 时间计算插入点：找到第一个起始时间 ≥ dropTime 的片段，在其前方插入。如果所有片段都小于 dropTime，追加到末尾。拖入过程中显示橙色落点指示线 |
| 拖入音频轨 | 从素材网格拖放音频到音频轨 | 如果音频轨已有片段：Toast 提示"请先删除当前音频后再添加"，拒绝拖入。如果音频轨为空：素材插入到起始时间 0 处 |

#### 8.5.8 缩放（Zoom）

- 触发：Ctrl/Cmd + 滚轮
- 范围：4-80 px/s
- 倍率：缩小 0.85，放大 1.18
- 效果：重渲染标尺 + 片段 + 播放头

#### 8.5.9 撤销/重做

- 历史栈：`HISTORY = { undo: [], redo: [], max: 30 }`
- pushHistory：深拷贝 CLIPS + 关键 S 状态字段（`t`, `zoom`, `sel`）到 undo 栈，清空 redo 栈，最多 30 条。audio 相关的 `volume`, `speed` 不纳入历史
- 触发时机：删除片段、分割片段、拖拽片段、导入素材
- undo：从 undo 弹栈 → 推入 redo → 恢复 CLIPS + `S.t`/`S.zoom`/`S.sel` → 重渲染
- redo：从 redo 弹栈 → 推入 undo → 恢复 CLIPS + `S.t`/`S.zoom`/`S.sel` → 重渲染

#### 8.5.10 字幕轨

- **字幕开关**：Timeline Toolbar 右侧新增「歌词字幕」Toggle 开关，默认开启。点击切换 `S.subtitleEnabled` 状态。
- **关闭效果**：字幕轨行变为 disabled（`opacity: 0.35, pointer-events: none`），视频预览区字幕浮层隐藏。
- **开启效果**：字幕轨行恢复正常交互，渲染字幕片段。
- **视频预览区浮层**：播放时根据当前播放头时间 `S.t` 查找对应字幕片段（遍历 `CLIPS['tr-sub']` 找到 `S.t` 所在的片段），有字幕时在视频底部中央显示（半透明黑色背景 + 白色文字，`video-subtitle-overlay`），无字幕时隐藏。
- **分割联动**：分割视频轨片段时，同时分割跨越播放头的字幕轨片段，保证字幕与视频对齐。

## 13. 异常与边界情况

### 13.1 时间轴边界

- **片段拖出轨道左侧：** 起始时间截断为 0，不出现负值
- **片段拖出轨道右侧：** 允许，但总时长 `S.dur` 自动扩展
- **缩放极限：** 最小 4px/s，最大 80px/s
- **片段最小宽度：** 20px，对应 `20 / zoom` 秒

### 13.2 删除与分割

- **未选中片段时点删除：** 无操作（按钮 disabled）
- **播放头不在任何片段上时分割：** 无操作或 Toast 提示
- **音频轨不允许分割：** v1 版本仅支持视频轨分割
- **字幕轨联动分割**：分割视频轨时，同时查找跨越播放头的字幕轨片段并一分为二，保持字幕与画面对齐

### 13.3 撤销/重做边界

- **历史记录上限：** 30 条，超过后最早的记录自动丢弃
- **清空历史：** 仅在首次 pushHistory 时确保 undo 栈为空

### 13.4 素材导入

- **不支持的文件类型：** Toast 提示"不支持的文件格式"
- **超大文件：** 单文件 ≤ 200MB，超出则 Toast 提示"文件大小超过 200MB 限制"
- **读取失败：** Toast "文件读取失败"，不追加到分类
- **音频轨替换规则**：音频轨仅支持单个音频片段。拖入新音频时如已有音频，拒绝并提示。用户需先删除旧音频才能替换。

### 13.5 播放

- **播放头超出总时长：** 循环回 0
- **拖拽进度条：** 播放状态不变（播放中继续播放，暂停中保持暂停）
