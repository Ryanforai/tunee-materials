L0 UI 模式层 —— "切"
控制全屏对话 ↔ 30% 对话 + 70% 画布的切换，是 Agent 识别意图后第一个触发的能力。
M1 activate_workspace P0
检测到 MV 意图后，全屏对话变为 30% 对话 + 70% 画布。初始化第一个 Page 和默认 region 布局（music_area / proposal_area / visual_assets / timeline）。这是整个产品的"入口门"。
输入 intent_type, initial_layout?
输出 { workspace_id }
L1 感知层 —— "读"
输入是用户上传的内容，输出是结构化理解。LLM 无法直接"看到"或"听到"，需要预处理。
P1 read_asset P0
读取用户上传素材的基本信息（URL、类型、大小、元数据）。
输入asset_id / URL
输出{ url, type, size, metadata }
存储 API
P2 analyze_music P0
分析音乐音频，提取 BPM、调性、情绪曲线、歌曲结构、能量曲线。
输入audio_url, lyrics_text?
输出{ bpm, key, duration, mood_curve, structure, energy_curve }
音频分析 API
P3 parse_lyrics P1
解析歌词文本，对齐时间戳，识别语言、段落结构（Verse/Chorus/Bridge）。
输入lyrics_text / lrc_url
输出{ language, lines: [{time, text}], structure }
文本解析 + LLM
P4 analyze_image P1
理解图片内容：描述、主色、构图、人脸、风格标签。用于"帮我找类似风格参考"场景。
输入image_url
输出{ description, colors, composition, faces[], style_tags }
VLM API
P5 analyze_video P1
理解用户上传的参考视频：时长、帧率、分辨率、场景分段、运动强度。
输入video_url
输出{ duration, fps, resolution, scenes[], motion_level }
VLM API + ffprobe
P6 read_text_file P2
读取用户上传的文本文件（剧本、分镜描述等），识别编码和语言。
输入text_url / text_content
输出{ content, encoding, language }
文件读取
L2 画布操作层 —— "放"
不产生新内容，只决定内容在哪里、以什么形式展示。前端操作，由 Agent 触发。push_to_canvas 是唯一的前端同步通道。
C1 push_to_canvas P0
将数据推送到画布的指定区域。唯一的前端同步通道 —— 所有工具产出都通过此通道推送。
输入event_type, target_region, data
输出{ success }
前端 EventBus
C2 open_page P1
在画布上新开一个 Page（类似浏览器新标签页），用于同时做多首 MV。
输入page_id, page_title, initial_content?
输出{ page_id }
C3 switch_page P1
切换到指定 Page，将画布焦点从一个 MV 项目转到另一个。
输入page_id
输出{ success }
C4 close_page P1
关闭一个 Page，释放对应画布工作区的资源。
输入page_id
输出{ success }
C5 list_pages P2
列出所有已打开的 Page，让 Agent 知道当前有哪些项目在进行。
输入—
输出[{ page_id, title, thumbnail, status }]
C6 focus_element P1
聚焦到画布上的某个元素 —— 缩放 + 居中，引导用户注意特定内容。
输入element_id
输出{ success }
前端定位
C7 update_element P1
更新画布上已有元素的部分属性（如替换视频 URL、修改标题）。
输入element_id, patch_data
输出{ success }
C8 remove_element P1
从画布移除元素，用户删除资产时同步清理。
输入element_id
输出{ success }
C9 layout_elements P2
在指定区域内按某种布局排列元素（网格、堆叠、时间线等）。视觉编排，非数据同步。
输入region, layout_type, elements
输出{ success }
L3 外部工具层 —— "做"
真正产生新内容的能力，需要调用外部 API 或系统。这些是注册到 LLM tools 数组的核心工具。
T1 generate_image P0
生成图片：角色参考图、场景图、分镜图、风格参考。支持 seed 控制随机性。
输入prompt, asset_type, model?, seed?, size?
输出{ image_url, seed, metadata }
预计 5-20s · 图片生成 API
T2 generate_video P0
生成视频片段。异步任务，返回 task_id 用于轮询状态。支持 frame_url 指定首帧。
输入video_prompt, frame_url?, duration, model?
输出{ video_url / task_id, status }
预计 60-120s · 视频生成 API
T3 run_ffmpeg P1
音视频处理：拼接多个视频片段、提取音频、转码、烧入字幕。
输入input_urls, operation, params
输出{ output_url, metadata }
预计 1-30s · ffmpeg
T4 evaluate_quality P0
质量检测：对生成的图片/视频评分，返回问题列表和改进建议。
输入asset_url, asset_type, criteria?
输出{ score, issues[], suggestion }
预计 3-10s · VLM API
T5 web_search P2
搜索参考素材（MV 风格、导演作品、视觉参考）。
输入query, source?, max_results?
输出{ references[] }
预计 2-8s · 搜索 API
T6 lip_sync P1
口型同步：将视频中的嘴型对齐到音频。歌词驱动 MV 场景的核心能力。
输入video_url, audio_url
输出{ video_url (synced), metadata }
预计 10-60s · Lip-Sync API
T7 face_swap P1
换脸 / 角色一致性：将参考面部特征迁移到目标视频，解决跨镜头"脸歪"问题。
输入source_video_url, face_image_url
输出{ video_url (swapped) }
预计 10-30s · Face-Swap API
L4 技能组合层 —— "编"
Skill = LLM 推理 + 多个 Tool 的编排。是"半成品工作流"，注册为 Agent Node 而非 LLM Tool。不直接暴露给 LLM 的 tools 数组。
编排型 Skill（6 个）
S1 generate_proposals P0
"帮我做个 MV"时触发。音乐分析 → Agent 构思 3 个方案 → 生成角色/场景图 → 汇总提案。
P2 analyze_music → LLM 构思 → S11/S12 × 6 → C1 push_to_canvas → 输出 proposals[]
S2 mv_planner P0
方案确定后，校验时间轴 → 撰写每个 SHOT 的 treatment（文字分镜描述）。
LLM 读取 mv_guide → 撰写 SHOT treatment → 输出文字分镜列表 → C1 push_to_canvas
S3 mv_generator P0
分镜确定后，将 treatment 转化为 video_prompt + frame_prompt，输出结构化 JSON。
treatment → LLM 构建 prompt → 输出 shot JSON[] → C1 push_to_canvas
S4 video_generation P0
逐 shot 调用视频生成 → 自动质检 → 推送到画布时间线。
逐 shot: T2 generate_video → T4 evaluate_quality → C1 push_to_canvas("video_clip_progress") → 全部完成 push_to_canvas("video_clips_ready")
S5 revise_shot P0
单镜头修改：定位目标 → 理解用户修改意图 → 重新生成 → 推送更新。
P1 read_asset → LLM 分析意图 → T1 generate_image → T2 generate_video → T4 evaluate_quality → C7 update_element
S6 export_mv P1
导出成片：拼接所有镜头 → 烧入字幕 → 转码为 MP4。
T3 ffmpeg(concat) → T3 ffmpeg(add_subtitles) → T3 ffmpeg(convert) → 输出 final video
创意分析 Skill（4 个模式）
S7 creative_analysis_story P0
故事模式：通过角色动作和故事推进让 MV 难忘，遵循现实物理规则。
音乐分析 + 用户意图 → LLM 叙事设计 → 输出 mv_guide JSON (story) → 时长 4-15s
S8 creative_analysis_visions P0
异世界模式：把角色放入不可能存在的世界，每帧都是前所未见的视觉奇观。
音乐分析 + 用户意图 → LLM 异世界设计 → 输出 mv_guide JSON (visions) → 时长 4-15s
S9 creative_analysis_lipsync P0
口型同步模式：按 video_model 分段规则生成镜头序列（infinite_talk / wan / seedance / kling）。
音乐分析 + 歌词 + 用户意图 → LLM 对口型设计 → 输出 mv_guide JSON (lipsync) → 按模型分段
S10 creative_analysis_clone P1
复刻模式：基于参考 MV 的视觉 DNA，重组到新歌时间轴上。
参考 MV 分析 + 新歌分析 → LLM 提取 DNA → 重组到新时间轴 → 输出 mv_guide JSON (clone)
资产生成 Skill（3 个）
S11 asset_generation_character P0
角色图提示词生成。基于 9 维度电影感框架产出 Midjourney / SD 兼容的 prompt。
主题 → 景别 → 构图 → 焦段+虚实 → 明暗骨架 → 冷暖阶级 → 逻辑验证 → 相机+胶片 → 神态+气质 → T1 generate_image
S12 asset_generation_scene P0
场景图提示词生成。同上 9 维度框架，聚焦场景构建（地点/氛围/光线/透视关系）。
主题 → 景别 → 构图 → 焦段+虚实 → 明暗骨架 → 冷暖阶级 → 逻辑验证 → 相机+胶片 → 神态+气质 → T1 generate_image
S13 cinematic_prompt_generator P1
电影感图片提示词总 Skill。适用于 MJ/SD 等工具，支持参考图反向分析。
描述/参考图 → 逐模块查阅词库 → 9 维度构建 prompt → 输出完整生成 prompt
视频生成 Skill 节点（4 个版本）
V1 mv-generation-一段式 P0
单节点完整版，接收创意分析的 mv_guide JSON，逐 shot 输出分镜脚本 JSON。
Unwrap → Normalize → Dispatch → 4 模型: infinite_talk / wan_video_2_7 / seedance_2_0 / kling_3_0_omni
V2 mv-generation-二段式 P1
planner + generator 双节点拆分，先输出文字 treatment 再生成 JSON。
planner: SUMMARIZE + DIRECTING + SHOT → generator: video_prompt + frame_prompt → 同 4 模型
SD2 mv-generation-sd2-一段式 P1
Seedance 2.0 专用单节点，仅支持 story/visions/clone，4-15s。
Unwrap → Normalize → Timeline Normalize → Rendering Family → Dispatch → seedance_2_0
SD2 mv-generation-sd2-二段式 P1
SD2 专用双节点，结构同主版本二段式但仅限 seedance_2_0。
planner → generator → seedance_2_0 专用渲染
