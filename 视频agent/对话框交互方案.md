# MV Agent 对话框交互方案

> 版本：v1.0 | 日期：2026-04-26 | 状态：草稿
> 上游文档：`MV Agent全貌：工具、交互与架构.md`、`Tunee MV Agent — 对话区产品需求.pdf`

---

## 一、整体定位

对话框是用户与 MV Agent 的唯一交互入口，承担三件事：

1. **接收指令** — 用户用自然语言表达创意意图
2. **呈现过程** — 工具调用的实时状态、进度反馈
3. **呈现结果** — 富内容卡片、资产预览、决策选项

对话框不操控画布，只通过 `push_to_canvas` 事件影响画布。

## 二、布局规范

```

│  Header（Agent 人格信息）    │  40px
├─────────────────────────────┤
│                             │
```

| 区域 | 规格 |

## 三、消息类型与视觉规范

### 3.1 用户消息（右对齐）

```
                   ┌──────────────────────┐
                   │  帮我为《夏日微风》  │
                   │  做一个 MV           │
                   └──────────────────────┘ [用户头像]
```

- 背景色：`oklch(0.580 0.180 48)`（琥珀橙）
- 文字色：white
- 圆角：12px 12px 2px 12px
- 支持 `[@Shot3]` 引用标签，标签样式：`background: oklch(0.9 0.05 48)` + 圆角 4px

---

### 3.2 TextReply（Agent 文字消息，左对齐）

```
[头像]  ┌──────────────────────────────┐
        │  角色图已生成，你可以选择喜  │
        │  欢的放入画布 ↗             │
        └──────────────────────────────┘
```

- 背景色：`oklch(0.96 0.00 0)`（近白灰）
- 文字色：`oklch(0.15 0.00 0)`
- 圆角：2px 12px 12px 12px
- Markdown 支持：`**bold**`、`- list`、`` `code` ``
- 流式输出：逐字渲染，光标闪烁 `|`，完成后光标消失

---

### 3.3 AssetCard（资产卡片）

```
[头像]  ┌──────────────────────────────┐
        │ [缩略图 288×162px 16:9]      │
        │─────────────────────────────│
        │  主角 · 角色图               │
        │  ─────────────────────────  │
        │  [放入画布]  [重新生成]  [↓] │
        └──────────────────────────────┘
```

- 缩略图：圆角 8px 8px 0 0，`object-fit: cover`
- 资产名 + 类型：`font-size: 13px`，`color: oklch(0.4 0.00 0)`
- 操作按钮区：高度 40px，三等分
  - 「放入画布」：accent 色填充
  - 「重新生成」：outline 样式
  - 下载图标：ghost 样式
- 入场动画：`scale(0.92) → scale(1.0)`，150ms ease-out

---

### 3.4 ProposalCard（方案卡片）

```
[头像]  ┌──────────────────────────────┐
        │  方案 A · 都市霓虹           │
        │  ─────────────────────────  │
        │  赛博朋克风格的城市夜景 MV   │
        │                              │
        │  Intro 雨夜街头              │
        │  → Verse1 霓虹灯下          │
        │  → Chorus 俯瞰城市          │
        │  ─────────────────────────  │
        │   [选择方案]    [查看详情]   │
        └──────────────────────────────┘
```

- 方案名：`font-weight: 600`，accent 色
- 分镜预览：最多 3 行，超出 `...` 收起，点击展开
- 「选择方案」按钮：点击后按钮变为 `✓ 已选择`，不可再次点击
- 多张 ProposalCard 连续出现时，间距收紧为 8px

---

### 3.5 OptionCard（决策选项卡片）

```
[头像]  ┌──────────────────────────────┐
        │  ⚠ 镜头 3 质量未通过         │
        │  面部区域不够清晰（42 分）   │
        │  ─────────────────────────  │
        │  ●  重新生成                 │  ← primary
        │  ○  调整提示词重试           │  ← secondary
        │  ○  保持现状                 │  ← ghost
        │  ○  手动修改                 │  ← ghost
        └──────────────────────────────┘
```

- 标题行：`⚠` 图标用 `oklch(0.65 0.15 80)`（黄橙警告色）
- 每个选项独占一行，高度 40px，左侧 radio 点
- 选中后：整张卡片锁定，选中项高亮，未选项变灰（`opacity: 0.4`）
- 选中的选项触发对应 `next_step` 工具调用

---

### 3.6 ProgressIndicator（进度卡片）

```
[头像]  ┌──────────────────────────────┐
        │  正在生成视频  3 / 8         │
        │  ████████░░░░░░░░░  37%     │
        │  ─────────────────────────  │
        │  ✓ Shot1  ✓ Shot2  ✓ Shot3  │
        │  ⟳ Shot4  · Shot5  · Shot6  │
        │  · Shot7  · Shot8           │
        └──────────────────────────────┘
```

- 进度条：accent 色，`height: 4px`，`border-radius: 2px`
- Shot 状态图标：`✓` 绿色完成 / `⟳` 旋转中（橙色）/ `·` 灰色待处理
- 进度数字和进度条实时更新（不重新渲染整个卡片，只 patch 数值）
- 全部完成后：标题变为「视频生成完成 ✓」，进度条填满 accent 色，2s 后缩起

---

### 3.7 LoadingState（思考中/调用工具中）

```
[头像]  ┌──────────────────────────────┐
        │  ● ● ●   AI 导演正在规划分镜  │
        │          预计 15 秒           │
        └──────────────────────────────┘
```

- 三点动画：依次弹跳，周期 1.2s
- 倒计时：`estimated_seconds` 开始倒数，到 0 后显示「正在努力中...」
- 替换规则：工具完成后，LoadingState 直接被结果卡片替换（不保留历史）

## 四、工具调用可视化

### 4.1 Tool Call 状态栏（底部固定条）

工具执行期间，输入区上方出现一条薄栏：

```
┌─────────────────────────────────────────┐
│  ⟳ generate_image · 角色图 · 已用 8s   [×]│
└─────────────────────────────────────────┘
```

- 高度 36px，背景 `oklch(0.96 0.02 48)`（浅琥珀）
- 显示：工具名、参数摘要（最多 20 字）、已用时间（秒计）
- `[×]` 点击可中止当前工具调用（触发 abort 信号）
- 多个并行工具：状态栏叠加显示，最多 3 行，超出折叠为「+ N 个工具运行中」

### 4.2 工具调用时序：用户可见层

```
用户发送消息
  ↓
[LoadingState] 出现（意图分析中）
  ↓
Tool Call 状态栏出现（工具名 + 计时）
  ↓
工具完成 → 状态栏消失
  ↓
结果卡片 / TextReply 流式出现
```

### 4.3 并行工具调用示意

当 `generate_image × 4` 并行时：

```
┌───────────────────────────────────────────┐
│  ⟳ generate_image · 主角         · 12s  │
│  ⟳ generate_image · 场景A        · 10s  │
│  ✓ generate_image · 场景B        · 9s   │
│  + 1 个工具运行中                         │
└───────────────────────────────────────────┘
```

全部完成后状态栏收起（height: 0, 200ms ease），结果卡片批量入场。

## 五、输入区设计

### 5.1 结构

```
┌─────────────────────────────────────────┐
│  [📎] [输入框，placeholder: 告诉我你的] │ [发送]
│       [想法，或 @ 引用画布元素...]      │
└─────────────────────────────────────────┘
```

- 输入框：`textarea`，min-height 40px，max-height 120px，自动增高
- 附件按钮：上传参考图（触发 `upload_asset` 前端操作）
- 发送按钮：有内容时 accent 色，无内容时灰色禁用
- 工具调用期间：输入框 `disabled`，发送按钮变为「停止」（红色）

### 5.2 @ 引用触发

输入 `@` 后弹出浮层选择器：

```
┌──────────────────────┐
│  @  [搜索框]         │
│  ─────────────────  │
│  [@Shot1] 海滩日落   │
│  [@Shot2] 城市夜景   │
│  [@方案A] 都市霓虹   │
│  [@主角] 角色图      │
└──────────────────────┘
```

- 选中后插入 `[@Shot1]` 标签 pill（可点击跳转画布）
- `[@xxx]` 在消息发出后渲染为带颜色的 badge

### 5.3 快捷指令

输入 `/` 触发命令面板：

| 命令 | 说明 |
|------|------|
| `/分析音乐` | 触发 `analyze_music` |
| `/生成方案` | 进入方案生成流程 |
| `/生成分镜` | 进入分镜规划 |
| `/生成视频` | 进入视频生成 |
| `/质量检测` | 对当前所有资产调用 `evaluate_quality` |
| `/搜索参考` | 弹出搜索框，调用 `web_search` |
| `/切换人格` | 打开人格选择 Sheet |

## 六、交互状态与边界处理

### 6.1 状态机

```
空闲（idle）
  → 用户发送消息 → 意图分析中
    → 无需工具 → 流式输出文字 → 空闲
    → 需要工具 → Tool Call 执行中
      → 成功 → 结果卡片出现 → 空闲
      → 失败 → 错误处理（见 6.2）
```

### 6.2 错误处理

| 场景 | UI 表现 | 用户操作 |
|------|---------|---------|
| 工具超时 | OptionCard：「重试 / 跳过 / 换模型」 | 三选一 |
| 工具返回错误 | ErrorBubble（红色边框）+ 错误摘要 | 「重试」按钮 |
| 网络断开 | 状态栏变红 + 「重连中...」 | 自动重连，失败后「刷新」 |
| 质量不达标 | OptionCard（OptionCard 3.5 所定义的） | 用户决策 |

ErrorBubble 样式：
```
[头像]  ┌──────────────────────────────┐
        │  ✕  生成失败                 │  ← 红色图标
        │  generate_video 超时（120s） │
        │  [重试]                      │
        └──────────────────────────────┘
```

### 6.3 消息历史与会话管理

- 单次会话内消息无限滚动，不分页
- 新消息出现时自动滚动到底部，但用户手动上翻后固定（出现「↓ 新消息」提示）
- 消息不持久化到本地（刷新即重置），仅 Agent state 持久化

## 七、动画总表

| 场景 | 属性 | 时长 | Easing |
|------|------|------|--------|
| 消息气泡入场 | `translateY(8px)→0` + `opacity 0→1` | 180ms | ease-out |
| AssetCard 入场 | `scale(0.92)→1` + `opacity 0→1` | 150ms | ease-out |
| OptionCard 展开 | `height 0→auto` | 140ms | ease-out |
| ProgressIndicator 更新 | 进度条 width 过渡 | 200ms | linear |
| Tool Call 状态栏出现/消失 | `height 0↔36px` | 160ms | ease |
| LoadingState 三点弹跳 | `translateY -4px→0` | 400ms/点 | ease-in-out |
| @ 浮层弹出 | `scale(0.95)→1` + `opacity` | 120ms | ease-out |
| 错误气泡抖动 | `translateX ±4px` 3次 | 320ms | ease-in-out |

## 八、与画布的交互边界

对话框**只做**：
- 展示 Agent 文字 / 卡片
- 接收用户输入
- 通过 `push_to_canvas` 触发画布更新

对话框**不做**：
- 直接操作 DOM 画布
- 持有画布元素的引用
- 提供画布编辑快捷键

画布 → 对话框的反向事件（如 `element_selected`）只更新 Agent state，不在对话框生成可见消息，除非 Agent 主动触发文字回复。

## 九、人格系统对对话框的影响

| 人格 | 语气 | 特殊 UI |
|------|------|---------|
| 专业导演 | 简洁、术语化 | OptionCard 选项用专业词 |
| 创意伙伴 | 活泼、多感叹号 | 气泡带随机 emoji 装饰点 |
| 温柔助手 | 暖、鼓励 | 完成时 confetti 微动画 |
| 毒舌审美 | 直接、带主观评价 | ErrorBubble 文案更犀利 |
| 艺术家 | 意象化语言 | 背景色调偏暖（`+0.01 chroma`） |

人格不影响工具调用逻辑和卡片数据结构，只影响文案和极少数视觉微调。
