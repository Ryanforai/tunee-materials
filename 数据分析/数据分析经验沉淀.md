## 一、报告写作原则

### 面向领导的报告结构（BLUF 原则）
> Bottom Line Up Front — 结论先行，数据支撑

```
1. 核心结论（1张对比表 + 1句提炼）
2. 分布数据（表格，不堆文字）
3. 漏斗/指标明细
4. 周环比变化
5. 关注点 & 建议（≤4条，编号列出）
```

**原则：**
- 用表格，不用段落堆砌
- 每个结论必须有对应数字支撑
- 建议要具体指向某个节点或问题，不写"持续关注"之类的废话
- 总字数控制在 600-800 字以内

## 二、漏斗分析方法

### 步进转化率 vs 累计转化率的验证方法

拿到漏斗图中的百分比，先判断是"步进率"还是"累计率"：

**验证公式（步进率）：**
```
整体转化率 ≈ 步骤1% × 步骤2% × 步骤3% × 步骤4%
```

**实例验证（来自本次任务）：**
- 非会员 <90s：73.74% × 41.12% × 29.8% × 65.07% ≈ **5.87%** → 实际值 5.68% ✓
- 会员 <90s：85.01% × 59.78% × 65.44% × 73.94% ≈ **24.55%** → 实际值 24.59% ✓

> 结论：图表中的百分比为**步进率**（每步相对上一步的通过率），而非累计转化率。

### 漏斗分析关注重点
1. **最大流失节点**：哪一步骤通过率最低
2. **用户分层对比**：会员 vs 非会员，短音乐 vs 长音乐，普通 vs lip_sync
3. **与总转化率的一致性**：用公式验证数据读取是否正确

## 三、从截图中读取数据的技巧

### 多张截图的整理顺序
1. 先识别每张图的**主题**（标题 + 图例）
2. 区分是**折线趋势图**（时间序列）还是**漏斗对比图**（阶段转化）
3. 优先读取**表格区域**（比图表精确），图表用于趋势感知

### 截图数据可信度判断
- 表格中的数字 > 图表上的视觉估读
- 整体转化率（Total Conv）是最可靠的锚点，用它来验证各步骤数据

### 常见截图信息提取清单
- [ ] 时间范围 & 是否有对比周期
- [ ] 图例颜色对应的分组（用户类型 / 时长 / mv_type）
- [ ] 表格中的 Average / Total 列
- [ ] 周环比的 Δ 值（绝对变化 pp）

## 四、MV 生成漏斗业务知识

### 标准漏斗节点（5步）
```
MV项目开启 → 创意节点确认 → 资产节点确认 → 分镜视频合成 → 视频下载
```

### 各节点业务含义
| 节点 | 含义 | 常见问题 |
|------|------|---------|
| 创意节点确认 | 用户选定创意方向 | 用户犹豫 / 选项太多 |
| 资产节点确认 | 用户确认素材（图片等） | 素材加载慢 / 交互体验差 |
| 分镜视频合成 | 点击开始生成 | 权益限制 / 等待体验差 |
| 视频下载 | 下载最终视频 | 质量不满意 / 入口不明显 |

### 各分组基准转化率（本次数据，可作后续对比参考）
| 分组 | 整体转化率 | 最大流失点 |
|------|-----------|-----------|
| 会员 普通 <90s | 24.59% | 资产节点（59.78%步进） |
| 会员 普通 ≥90s | 23.49% | - |
| 非会员 普通 <90s | 5.68% | 资产节点（41.12%步进） |
| 非会员 普通 ≥90s | 2.0% | - |
| 会员 lip_sync <90s | 2.75% | - |
| 非会员 lip_sync <90s | 1.17% | - |

> 以上数据为 2026-04-08 至 04-15 基准值，后续对比可参考。

## 五、常用分析维度 Checklist

每次做 MV 相关数据分析时，检查以下维度是否覆盖：

- [ ] 用户类型分层（会员 / 非会员）
- [ ] 音乐时长分层（<90s / ≥90s）
- [ ] mv_type 分层（普通 / lip_sync）
- [ ] 整体转化率
- [ ] 各漏斗步进率
- [ ] 周环比（Δpp）
- [ ] 日均启动量趋势
