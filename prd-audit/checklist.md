# PRD 通用检查清单

> **使用方法**：按 5 个核心问题逐项勾选。**在 PRD 里找不到答案的就是缺失项**。严格标准——宽松的审查没有价值。  
> **范围**：这份清单覆盖通用审查。Agent 类产品还需要再过 `checklist-agent.md`。

---

## 问题 1：为什么做？

### 问题和证据
- [ ] 明确说出具体问题（不是"提升体验"这种虚词）
- [ ] 有证据支持（数据、用户反馈、访谈、竞品）
- [ ] 不做会怎样（帮助判断优先级）

### 用户
- [ ] 目标用户是谁（具体到分层：免费 / 付费、T1 / T2 / T3、新 / 老）
- [ ] 核心场景是什么（具体到动机）

### 成功指标
- [ ] 指标清晰可测量（不是"满意度提升"）
- [ ] 有现状基线值
- [ ] 有目标值 + 达成时间
- [ ] 北极星指标 + 辅助指标

---

## 问题 2：具体做什么？

### 范围
- [ ] **In scope**：本期要做什么（清单）
- [ ] **Out of scope**：本期**不做**什么（清单）
- [ ] 后续版本计划：放到 V2 / V3 的是什么

### 和现有功能的关系
- [ ] **共存**：哪些功能并存、入口如何区分
- [ ] **互斥**：哪些功能不能同时用、冲突时谁优先
- [ ] **替换**：替换了哪些旧功能、旧入口是否下线、灰度策略

---

## 问题 3：正常情况怎么走？

- [ ] **从哪里进入**：入口位置、触发条件
- [ ] **前置状态**：用户要具备什么条件
- [ ] **主流程**：每一步用户操作 → 系统响应 → 状态变化
- [ ] **关键字段**：输入什么、展示什么、存储什么
- [ ] **结束状态**：流程完成后用户看到什么
- [ ] **至少一组完整示例**（跑通全流程）
- [ ] **流程图 / 状态图**（复杂流程必备）

---

## 问题 4：不正常怎么办？

### 输入边界
- [ ] 空值（什么都不填 / 选了又清空）
- [ ] 超长（字段字符数上限）
- [ ] 特殊字符（emoji、标点、控制字符）
- [ ] 多语言输入（中英混合、RTL）
- [ ] 非法格式（邮箱、手机号、URL）
- [ ] 恶意输入（XSS、注入）

### 状态冲突
- [ ] 并发操作（同用户多设备）
- [ ] 重复提交（快速连点、网络重发）
- [ ] 过期数据（打开后数据被改）
- [ ] 中途退出（未完成就关闭）

### 外部依赖（如有，详见 checklist-agent.md C 区）
- [ ] 超时（阈值多少、怎么表现）
- [ ] 失败（提示什么、能不能重试、**重试是否消耗配额**）
- [ ] 限流（降级策略）
- [ ] 完全宕机（有无兜底）

### 权限边界
- [ ] 未登录能用吗、能用到哪一步
- [ ] 无权限用户（未开通会员）的表现
- [ ] 被封号 / 限权用户的表现
- [ ] 不同角色差异

### 数量边界
- [ ] 第一次用（空状态长什么样）
- [ ] 0 条数据 / 无搜索结果
- [ ] 达配额上限
- [ ] 批量过多
- [ ] 历史数据过多（分页 / 性能）

### 业务边界
- [ ] 免费 vs 付费差异
- [ ] 国家 / 地区差异（合规、支付）
- [ ] 新 vs 老用户差异（引导、默认值、权益）
- [ ] 未成年人特殊处理（如需）

### 计费（重要）
- [ ] 什么动作扣费
- [ ] 失败扣不扣费
- [ ] 重试扣不扣费
- [ ] 退款规则
- [ ] 灰度期间怎么算费

---

## 问题 5：怎么知道做成了？

### 指标定义
- [ ] 成功指标定义完整（分子 + 分母 + 时间窗）
- [ ] 有基线值和目标值
- [ ] 数据源明确（哪个事件、哪个字段）

### 埋点
- [ ] 关键路径每一步都有埋点
- [ ] 事件命名符合规范
- [ ] 事件参数完整（支持分层分析：用户类型、地区、版本等）
- [ ] 漏斗能走通（入口到转化无断点）
- [ ] 失败事件有专门埋点（不只是成功事件）

### 实验和报表
- [ ] 是否需要 A/B 实验（实验方案、切流量比例）
- [ ] 报表需求（谁看、看什么、多久看一次）
